Формулы рандомизации

Простыми словами: при обратном выборе без сглаживания система исключает участников много раз подряд, поэтому математика «усиливает» превосходство сильных: участники с большими весами заметно чаще доходят до финала, а слабые чаще выбывают. При сглаженном обратном выборе этот эффект смягчается так, что итоговые шансы на победу становятся практически такими же, как при прямом выборе.

Обозначения

  • Для каждого участника i задан исходный вес w_i (если задано 0 или отрицательное, берётся минимум 1).
  • Эффективный вес: w*_i = max(1, w_i).

Прямой выбор (один победитель)

При прямом типе события выбирается победитель одним жребием со взвешенной вероятностью.

Вероятность выбора участника i:

P(выбран i) = w*_i / (Σ_j w*_j)

То есть чем больше вес участника, тем выше его шанс быть выбранным победителем.

Обратный выбор без сглаживания (EventType.reverse)

Система по очереди исключает участников, пока не останется один — он и является победителем. Вероятности выбывания считаются на основе инверсных весов с показателем α = 1:

Пусть w*_i = max(1, w_i), а Σ = Σ_k w*_k по активным участникам.
Инверсный вес:   w'_i ∝ Σ / (w*_i)^α,  где α = 1
Нормировка:      P(выбывает i на шаге) = w'_i / (Σ_{k в активных} w'_k)
  • Больший исходный вес означает меньший риск выбыть на каждом шаге.
  • Из‑за большого числа раундов это математически усиливает преимущество сильных: они чаще доходят до финала, чем при прямом выборе.

Обратный сглаженный (EventType.reverse_soft)

Используется та же схема по выбыванию, но инверсия весов сглаживается показателем α = 0.5:

Инверсный вес:   w'_i ∝ Σ / (w*_i)^α,  где α = 0.5
Нормировка:      P(выбывает i на шаге) = w'_i / (Σ_{k в активных} w'_k)
  • Сглаживание уменьшает «перекос» в пользу сильных.
  • Итоговые шансы на победу близки к шансам при прямом выборе.
  • Масштабный множитель (например, ×100 и округление до целого) не влияет на вероятности — он используется только для удобства целочисленного выбора.

После исключения участника пересчитываются активный список, сумма Σ и, соответственно, инверсные веса w'_i. Процесс повторяется до одного оставшегося участника.

Связь с реализацией

В коде приложения (app/app.py) параметр α принимается равным 1 для «Обратного» и 0.5 для «Обратного сглаженного». Основные шаги:

  1. Подготовка пар (id, вес) для активных участников.
  2. Функция взвешенного выбора по равновероятному целому числу r в диапазоне [1, Σ max(1, w)]: 
    # Упрощённо
r ∈ {1, 2, ..., T}, где T = Σ_j max(1, w_j)
Накопленная сумма S ← 0
Для каждого участника (id, w):
    S ← S + max(1, w)
    если r ≤ S: выбрать этого участника
  3. Для обратных типов перед вызовом взвешенного выбора формируются инвертированные веса по соответствующей формуле (α = 1 или α = 0.5), затем выбранный участник помечается как «выбыл».

Примеры

Пусть даны три участника с весами: A=1, B=2, C=5. Тогда:

  • Прямой выбор: Σ=1+2+5=8. P(A)=1/8, P(B)=2/8, P(C)=5/8.
  • Обратный сглаженный (α=0.5): Σ=8. Инверсные веса пропорциональны Σ/√w: A'≈8/1=8, B'≈8/√2≈5.66, C'≈8/√5≈3.58. На первом шаге вероятности выбывания пропорциональны этим значениям: P(выбывает A)≈8/(8+5.66+3.58)≈0.46, P(выбывает B)≈0.33, P(выбывает C)≈0.21.
  • Обратный без сглаживания (α=1): Σ=8. Инверсные веса пропорциональны Σ/w: A'≈8, B'≈4, C'≈1.6. Вероятности выбывания на первом шаге: P(A)≈8/(8+4+1.6)≈0.57, P(B)≈0.29, P(C)≈0.11 — сильный участник ещё реже выбывает.

Таким образом, при α=1 преимущество сильных выражено сильнее, чем при α=0.5. Масштабирование на целые числа (×100 и округление) не меняет этих вероятностей.
